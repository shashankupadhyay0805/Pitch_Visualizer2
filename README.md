## The Pitch Visualiser

The Pitch Visualiser is a FastAPI web app that converts a short narrative pitch into a visual storyboard. It:

- Segments your input text into at least three logical scenes.
- Uses an LLM (Gemini) to engineer rich, cinematic prompts for each scene.
- Calls a text-to-image model on Hugging Face to generate one image per prompt.
- Displays all scenes in a single dynamic storyboard page.

---

## 1. Features & Capabilities

- **Text input**: Paste a paragraph (3–5+ sentences) describing your idea.
- **Narrative segmentation**: The app splits the text into sentence-level scenes, with a fallback split if needed.
- **Prompt engineering with Gemini**:
  - Expands each sentence into a detailed visual description.
  - Enforces a consistent visual style (cinematic, digital illustration).
- **Image generation with Stable Diffusion XL**:
  - Sends engineered prompts to `stabilityai/stable-diffusion-xl-base-1.0` via the Hugging Face Inference API.
  - Saves each generated image to `static/images/*.png`.
- **Storyboard presentation**:
  - Renders a responsive grid of panels.
  - Each panel shows the generated image, the original text segment, and the exact prompt used.
- **User-selectable visual style**:
  - Dropdown lets users choose from several styles (Digital Art, Photorealistic, Watercolor, Comic Book); this is appended to every prompt.

## 3. API Key Management

This project uses two API providers:

- **Gemini (Google Generative AI)** for text-to-text prompt refinement.
- **Hugging Face Inference API** for text-to-image generation.

### 3.1. Configure Environment Variables

Create a file `.env` in the project root (same folder as `app/` and `requirements.txt`) with:

```env
GEMINI_API_KEY=your_gemini_api_key_here
HF_API_KEY=your_huggingface_token_here
```

This file is loaded by `python-dotenv` at startup.

#### 3.1.1. Getting a Gemini API key

1. Go to Google AI Studio and create a new API key.
2. Copy the key value.
3. Paste it into `GEMINI_API_KEY` in `.env`.
4. If a key is ever reported as leaked or disabled, revoke it in AI Studio and generate a new one.

#### 3.1.2. Getting a Hugging Face token

1. Log in to Hugging Face.
2. Navigate to **Settings → Access Tokens**.
3. Create a **new token** with:
   - **Type**: `Read` (fine-grained, read-only is sufficient).
4. Copy the token and paste it into `HF_API_KEY` in `.env`.
5. If the token expires or is compromised, revoke it from the same page and create a fresh one.

> **Note:** Never commit `.env` to version control or share your keys publicly.

## 5. Design Choices & Prompt Engineering

### 5.1. Narrative Segmentation

File: `app/segmenter.py`

- Primary segmentation uses NLTK’s `sent_tokenize` to obtain sentence-level scenes.
- If NLTK data is unavailable or the result has fewer than three segments, the function falls back to splitting on `"."`.
- This ensures that even unconventional text still yields at least three panels, meeting the assignment requirement.

### 5.2. Prompt Engineering with Gemini

File: `app/prompt.py`

- A fixed style anchor string:

  ```python
  STYLE = "digital illustration, cinematic lighting, concept art"
  ```

- For each sentence + user-selected style, the app builds an instruction prompt asking Gemini to:
  - Convert the sentence into a detailed **cinematic visual scene**.
  - Maintain the chosen artistic style.
- The response text (if present) is used directly as the image prompt.
- On error or empty output, the app falls back to:

  ```text
  original sentence + ", " + STYLE
  ```

This design guarantees a usable prompt even if the LLM call fails.

### 5.3. Image Generation

File: `app/image_generator.py`

- Uses `huggingface_hub.InferenceClient` with the configured `HF_API_KEY`.
- Sends each engineered prompt to:

  ```text
  stabilityai/stable-diffusion-xl-base-1.0
  ```

- Saves the returned image locally under `static/images/{uuid}.png` and returns the relative path.
- If any exception occurs (e.g., 401/403 from Hugging Face), the function logs the error and returns a fallback `static/images/error.png`.

### 5.4. Storyboard Presentation

- The FastAPI backend exposes:
  - `GET /` → serves `templates/form.html`.
  - `POST /generate` → accepts JSON `{ "text": ..., "style": ... }` and returns a JSON `storyboard` list.
- The front-end JavaScript:
  - Shows a loading spinner during generation.
  - Renders each scene as a card with image + text + prompt.
  - Uses responsive CSS grid for a clean, modern layout.
